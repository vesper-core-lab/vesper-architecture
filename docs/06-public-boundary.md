# Public Boundary

This repository is a public architecture and documentation repository for Vesper.

It is intended to share non-sensitive system design notes, public roadmap, synthetic examples, and development logs.

## What This Repository Is

This repository is:

- A public architecture note collection
- A design principle archive
- A non-sensitive roadmap
- A place for synthetic examples
- A public build log for early project communication

## What This Repository Is Not

This repository is not:

- The full Vesper implementation
- A production deployment repository
- A customer project repository
- A commercial workflow repository
- A private research archive
- A place for raw interviews or confidential materials

## What Remains Private

The following materials remain private:

- Runtime implementation
- Production configuration
- API keys and tokens
- Server paths
- Internal business documents
- Raw interview records
- Customer data
- Commercially sensitive workflows
- Detailed internal execution logic

## Why the Boundary Exists

Vesper is still under active prototyping.

The public repository is used to communicate architecture direction without exposing unstable implementation details or sensitive business information.

## Long-Term Direction

As the architecture becomes more stable, selected examples, templates, SDK concepts, and module patterns may be gradually published.
