# Vesper Public Roadmap

This roadmap is a public, non-sensitive overview.

It does not include private implementation details.

## Phase 0 — Concept and Architecture

- Define Vesper as an AI Agent Operating System
- Establish governance-first design principles
- Define event-driven architecture direction
- Define object-based operations direction

## Phase 1 — Core Prototype

- Input adapter prototype
- Task system
- Object system
- SkillBook
- Natural language console
- Working memory
- Basic persistence

## Phase 2 — Object System v2

- Creator object
- Product object
- Campaign object
- Video object
- Order object
- Enhanced report object
- Task-object relationships
- Object persistence and listing consistency

## Phase 3 — Industry Module Prototype

Initial focus:

- MCN operations
- Cross-border e-commerce workflows
- Creator / Product / Campaign / Video / Order / Report operations
- Structured reporting
- Operational task tracking

## Phase 4 — Governance and Memory

- Approval engine
- Guardrail / Governor
- Execution review
- Feedback memory
- Audit trail
- Human-in-the-loop workflows

## Phase 5 — SDK and Rules Layer

- Industry rule configuration
- Module templates
- Developer-facing examples
- Public documentation
- Controlled extension interface
